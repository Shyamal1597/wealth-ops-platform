🔧 Fix IIS Reverse Proxy Configuration

⚠️ Current Issue:
Your website shows a 500 Internal Server Error because IIS Application Request Routing (ARR) is either not installed or not enabled.

Solution: Enable ARR Proxy

✅ Option 1: Run the Batch File (Easiest)

1 Locate the file:
C:\Users\SSFL-RETAIL-017\sunidhi-nextjs\ENABLE_ARR_PROXY.bat
2 Right-click on the file and select "Run as administrator"
3 Click "Yes" when Windows asks for permission (UAC dialog)
4 Follow the on-screen instructions

🔄 Option 2: If Option 1 Fails (ARR Not Installed)

If the batch file shows an error that ARR is not installed, you need to install it first:

1 Download ARR 3.0 from Microsoft:
Visit: https://www.iis.net/downloads/microsoft/application-request-routing
2 Install ARR 3.0 by running the downloaded installer
3 After installation, run the ENABLE_ARR_PROXY.bat file again (as administrator)

✨ After Success:

Your website will be accessible at:

  • Intranet URL: http://192.168.48.102
  • Test URLs:
    • http://192.168.48.102/about/story
    • http://192.168.48.102/login
    • http://192.168.48.102/feedback

🔍 What This Does:

  • Enables IIS Application Request Routing (ARR) proxy functionality
  • Allows IIS to forward requests from port 80 to your Next.js server on port 3000
  • Makes your website accessible without the :3000 port number
  • Restarts IIS to apply the changes

📝 Notes:

  • Your Next.js server must be running on port 3000 for this to work
  • If your colleagues can't access the site, check Windows Firewall settings
  • The website is currently configured for intranet access at IP: 192.168.48.102
